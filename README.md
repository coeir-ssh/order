## 📊 분석 요약

### 시트 구조
- **8개 탭** 파이프라인: `raw 데이터 → 발주량_계산(엔진) → 메인(대시보드)`
- **983행 SKU** 운영

### 핵심 산식 (발주량_계산 탭)
| 컬럼 | 의미 | 수식 |
|---|---|---|
| P | 1일 평균 판매량 | 최근 1개월 raw / 30 |
| R | 발주 1차 산출값 | `MAX(0, ROUNDUP((60+생산기간)×일평균))` |
| S | 미니멈 반영 | `MAX(미니멈, R)` |
| U | 판매 가능일수 | `(창고재고-추가수량) / 일평균` |
| V | 발주 D-day | `판매가능일수 - 생산기간` |
| Y | 실제 발주수량 | `MAX(S-창고재고, 0)` + MOQ 가드 |
| AA | 박스환산 발주량 | `박스환산 × 박스당 수량` |

### 발견된 이슈 (9건)
- 🔴 **심각 (3)**: 시각적 우선순위 부재 / 체크박스 수동 의존 / 알림 부재
- 🟠 **중간 (4)**: 쿠팡재고 미반영 / 추가수량 만료일 없음 / 생산기간 일률 적용 / 시즌성 미반영
- 🔵 **낮음 (2)**: 휴무일 미연동 / 검증 셀 부재

### 개선 로드맵
1. **시트 재구성** — 자동 체크 로직, 휴무일 연동, 검증 셀, 시즌성 보정
2. **외부 가시화** — 슬랙/카톡 알림 봇, 모바일 대시보드, 발주서 자동 생성
